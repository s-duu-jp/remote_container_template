# make-projects: プロジェクト初期構成

プロジェクト全体の初期構成を順番に生成するオーケストレーター。
**最初に必要な質問をすべてまとめて行い**、その後サブスキルを順次呼び出す。

---

## STEP 0: 質問の収集（先に全部聞く）

タスク登録は**質問完了後**に行う。質問内容によっては登録不要なタスクがあるため。

各質問は1問ずつ提示し、回答を得てから次に進む。

### Q1. リポジトリ名は何にしますか？

```
GitHubリポジトリ名を教えてください（例: my-project）
```

→ `repo_name` を取得（必須）

### Q2. リポジトリの公開範囲は非公開で良いですか？

```
非公開（private）で良いですか？（yes / no）
※ yes → private
※ no → public
```

→ `repo_visibility` = `private` または `public`

### Q3. バックエンドを作成しますか？

```
バックエンドを作成しますか？（yes / no）
```

- yes → 続けて Q3-1 を質問する
- no → スキップ

#### Q3-1. バックエンドのアプリ名は何にしますか？（Q3でyesの場合のみ）

```
バックエンドのアプリ名を教えてください（例: manage, api, admin）
```

→ `backend_app_name` を取得

### Q4. フロントエンドを作成しますか？

```
フロントエンドを作成しますか？（yes / no）
```

- yes → 未実装である旨を伝える
- no → スキップ

### Q5. インフラストラクチャを作成しますか？

```
インフラストラクチャを作成しますか？（yes / no）
```

- yes → 未実装である旨を伝える
- no → スキップ

### 確認

全回答を表でまとめて提示し、承認を得る：

```
| 項目 | 値 |
|------|-----|
| リポジトリ作成 | 名前: xxx, 公開設定: private/public |
| バックエンド作成 | yes（アプリ名: xxx） / no |
| フロントエンド作成 | yes（未実装のためスキップ） / no |
| インフラ作成 | yes（未実装のためスキップ） / no |
```

---

## STEP 1: タスク登録（質問結果に応じて）

ユーザー承認後、`TaskCreate` で以下を登録する。**「作成する」と回答したものだけ登録する**：

| 条件         | タスク名                                | activeForm             |
| ------------ | --------------------------------------- | ---------------------- |
| 常に登録     | ユーザーへの質問（completed状態で登録） | 質問を実施             |
| Q1で名前あり | リポジトリの作成                        | リポジトリを作成中     |
| Q3=yes       | バックエンドの作成                      | バックエンドを作成中   |
| Q4=yes       | フロントエンドの作成                    | フロントエンドを作成中 |
| Q5=yes       | インフラの作成                          | インフラを作成中       |

依存関係: リポジトリ作成 → バックエンド作成 → フロントエンド作成 → インフラ作成 の順で `addBlockedBy` を設定。

---

## STEP 2: リポジトリの作成

`make-projects-repository` スキルを読み込んで実行する。

**親から引き継ぐパラメータ:**

- `project_name` = `repo_name`
- 公開設定 = `repo_visibility`

実行後、以下を後続STEPに引き継ぐ：

- `project_name`
- `owner`（`gh api user --jq .login` で取得）
- `module_prefix` = `github.com/{owner}/{project_name}`

完了後タスクを `completed` にする。

---

## STEP 3: バックエンドの作成

Q3で「yes」の場合のみ実行。
`make-projects-backend` スキルを読み込んで実行する。

**親から引き継ぐパラメータ:**

- `app_name` = `backend_app_name`
- `module_prefix`（STEP 2で取得した値）

完了後タスクを `completed` にする。

---

## STEP 4: フロントエンドの作成

🔲 **未作成** — Q4で「yes」だった場合は未実装である旨を伝える。

---

## STEP 5: インフラの作成

🔲 **未作成** — Q5で「yes」だった場合は未実装である旨を伝える。

---

## 重要なルール

- 質問はすべてSTEP 0でまとめて行う。サブスキル実行中は追加質問しない
- タスク登録はSTEP 1（質問完了後）で行う。質問前には登録しない
- サブスキルが直接呼ばれた場合（親経由でない）はサブスキル側で必要な質問を行う
